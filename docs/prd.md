# 產品需求文件 (PRD): HoldYourBeer

## 1. 專案願景 (Vision)

打造一款簡潔、直觀的啤酒品飲追蹤應用，讓啤酒愛好者可以輕鬆記錄、管理並回顧自己的品飲歷程，從而深化對啤酒文化的探索與樂趣。

## 2. 產品概述 (Overview)

**HoldYourBeer** 是一款專為啤酒愛好者設計的追蹤工具。使用者可以記錄他們品嚐過的每一款啤酒的品牌和名稱，應用程式會自動計算每款啤酒的品飲次數。此外，使用者還可以為每次品飲添加筆記，建立個人化的品飲資料庫。

本專案包含一個核心的 Web 應用程式以及一個對應的 Flutter 行動應用程式。

## 3. 目標使用者 (Target Audience)

*   **啤酒新手**：希望探索不同啤酒、記錄喜好以建立品味地圖的使用者。
*   **資深啤酒愛好者**：需要系統化工具來管理大量品飲記錄、比較不同酒款風味的專業玩家。
*   **社交分享者**：喜歡在社群中分享自己品飲體驗的使用者。

## 5. 非功能性需求 (Non-Functional Requirements)

| 類別 | 需求 |
| :--- | :--- |
| **技術棧** | **後端**: Laravel 12, PHP 8.3<br>**前端**: Livewire (Web)<br>**資料庫**: PostgreSQL 17 (開發), SQLite (測試)<br>**開發環境**: Laradock |
| **開發流程** | 遵循**規格驅動開發 (Spec-Driven Development)**，所有功能需先在 `/spec` 目錄下定義 Gherkin 特徵檔案與 OpenAPI 規格。 |
| **UI/UX** | 採用 **Mobile-First** 響應式設計，使用 Tailwind CSS 確保在不同尺寸裝置上都有一致且良好的體驗。 |
| **程式碼品質** | 所有程式碼提交前，必須通過 PHPUnit 單元測試與功能測試，並由 PCOV 產生覆蓋率報告。 |

## 6. 系統架構與資料模型

*   **Web 應用**: 基於 Laravel 框架，採用 Livewire 處理前端互動，實現動態的單頁應用體驗。
*   **資料庫**: 主要包含 `users`, `beers`, `brands`, `tastings` 等資料表，用以儲存使用者資訊、啤酒品項、品牌以及每一次的品飲記錄。
*   **API**: 遵循 OpenAPI 3.0 規範，確保前後端分離與行動應用整合的順暢。

## 7. 未來展望 (Out of Scope / Future)

*   **社交功能**：好友系統、分享品飲記錄到社群媒體。
*   **數據視覺化**：提供圖表分析使用者的品飲偏好，如品牌分佈、風味類型佔比等。
*   **酒廠與商家資訊整合**：提供啤酒的購買地點或酒廠的詳細介紹。
*   **進階搜尋與過濾**：根據評分、風味標籤、產地等條件篩選啤酒。
